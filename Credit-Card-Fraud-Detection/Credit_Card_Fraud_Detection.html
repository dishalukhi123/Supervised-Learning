Module 01 / Exploration

Executive Summary & High-Fidelity Dataset Profiling

In real-world payment networks processing millions of operations every minute, financial anomalies and fraudulent vectors represent an incredibly tiny subsection of overall system traffic. In this dataset, out of 284,807 transaction events, exactly 492 occurrences are validated as malicious fraud vectors.

Standard metric architectures like accuracy fail spectacularly here; a baseline model predicting 100% legitimate states scores a deceptive 99.83% accuracy while missing every single active threat. Therefore, this project emphasizes area under the Precision-Recall curve (PR-AUC) and operational cost matrices over naive metric optimizations.

Module 02 / Engineering

Mathematical Feature Transformations & Class Resampling

Raw monetary amount parameters show heavy right-skewed tail distributions with immense variances. To stabilize model parameters and streamline gradient convergence steps, our pipeline enforces strict data mathematical transformations.

Core Pipeline Mathematical Operations
Amount\_Scaled = \ln( ext{Amount} + 1)
ext{Temporal Hour Extracted} = ( ext{Time} \pmod{86400}) \div 3600
Multi-Strategy Class Imbalance Resampling Lifecycle

To prevent structural metrics from overfitting to the majority class distribution, three dedicated resampling paradigms were integrated during the model training cycle:

Random Undersampling (RUS) Execution
Reduces majority records to balance the localized representation index up to precisely 10% target density.
SMOTE Synthetic Neighbor Oversampling
Synthesizes minority data coordinates inside K-Nearest Neighbor regions to eliminate structural algorithmic bias.
Dynamic Cost-Weighted Penalization
Embeds class frequencies directly into the objective loss matrix, penalizing false negatives severely during updates.
Mandatory Test Partition Separation Protocol
All test arrays (X_test, y_test) are intentionally kept outside the resampling cycle, preserving their native 0.17% distribution properties to ensure unbiased validation results.
